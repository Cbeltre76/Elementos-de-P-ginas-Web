Temas de Desarrollo Web

Tema 1: Elementos de Páginas Web

  • I. Orígenes de la Web
    • 1. Web 2.0

      La Web 2.0 se caracteriza por la participación activa de los usuarios, permitiendo la creación y compartición de contenido en plataformas colaborativas como blogs y redes sociales.

    • 2. Web 3.0

      La Web 3.0, o Web Semántica, busca mejorar la comprensión de la información por parte de las máquinas a través de datos estructurados y ontologías, facilitando una interacción más inteligente.

    • 3. La W3C y los estándares

      La World Wide Web Consortium (W3C) desarrolla estándares para asegurar la interoperabilidad y calidad de la web, como HTML, CSS, y XML.

  • II. Lenguajes de etiquetas
    • 1. SGML

      El Standard Generalized Markup Language (SGML) es un lenguaje de marcado que proporciona un marco para la definición de lenguajes de marcado como HTML y XML.

    • 2. XML

      Extensible Markup Language (XML) es un lenguaje de marcado que se utiliza para definir y estructurar datos de manera que sea legible tanto para humanos como para máquinas.

    • 3. HTML

      Hypertext Markup Language (HTML) es el estándar principal para la creación de páginas web, definiendo la estructura y contenido de las páginas mediante etiquetas.

    • 4. XHTML

      Extensible Hypertext Markup Language (XHTML) es una versión más estricta de HTML que usa la sintaxis de XML para garantizar una mayor uniformidad y precisión en el marcado.

    • 5. HTML5

      HTML5 es la última versión de HTML, que incluye nuevas características como elementos semánticos, audio, video y gráficos para mejorar la funcionalidad y la accesibilidad de las páginas web.

  • III. Otros lenguajes relacionados al desarrollo web
    • 1. CSS/CSS3

      CSS (Cascading Style Sheets) es un lenguaje utilizado para describir la presentación de un documento HTML. CSS3 añade nuevas características como transiciones y animaciones.

    • 2. JavaScript

      JavaScript es un lenguaje de programación que permite añadir interactividad y dinamismo a las páginas web, ejecutándose en el navegador del cliente.

    • 3. Ajax

      Asynchronous JavaScript and XML (Ajax) es una técnica para actualizar partes de una página web sin recargarla completamente, permitiendo una experiencia de usuario más fluida.

    • 4. Tecnologías y lenguajes del lado del servidor

      Estos incluyen lenguajes como PHP, Python y Ruby que se ejecutan en el servidor para manejar la lógica de backend y generar contenido dinámico para los usuarios.

    • 5. Navegadores

      Los navegadores web interpretan y renderizan el código HTML, CSS y JavaScript. Ejemplos incluyen Chrome, Firefox y Safari, cada uno con sus propias características y compatibilidades.

    • 6. Motores renderizados

      Los motores renderizadores, como Blink y Gecko, son responsables de procesar el código web y mostrar el contenido visualmente en el navegador.

  • IV. Herramientas de desarrollo
    • 1. Software WYSIWYG

      Los editores WYSIWYG (What You See Is What You Get) permiten diseñar páginas web visualmente sin necesidad de escribir código manualmente, como Adobe Dreamweaver.

    • 2. Otras alternativas
      • i. Figma

        Figma es una herramienta de diseño colaborativo basada en la web que permite crear prototipos y diseños de interfaces con edición en tiempo real.

      • ii. InVision

        InVision facilita la creación de prototipos interactivos y la colaboración entre diseñadores y desarrolladores para visualizar el diseño final.

      • iii. Zeplin

        Zeplin proporciona una plataforma para la colaboración entre diseñadores y desarrolladores, permitiendo la exportación y documentación de estilos y especificaciones de diseño.

      • iv. GitHub

        GitHub es una plataforma de control de versiones y colaboración que permite la gestión del código fuente y facilita el trabajo en equipo mediante repositorios.

    • 3. Inspección de código desde el navegador

      La inspección de código en el navegador permite ver y modificar el HTML, CSS y JavaScript de una página web en tiempo real utilizando las Herramientas de Desarrollador.

Tema 2: El Diseño De Una Página Web

  • I. Tipos de páginas Web

    Los tipos de páginas web incluyen páginas estáticas, dinámicas y aplicaciones web. Cada tipo tiene diferentes propósitos y estructuras según la necesidad del contenido y la interactividad.

  • II. Puntos básicos en la elaboración de una página Web

    Al elaborar una página web, es crucial considerar el diseño visual, la usabilidad, la accesibilidad y la optimización para diferentes dispositivos y navegadores.

Tema 3: Estructura De Una Página Web

  • I. Jerarquía de directorio de un proyecto web

    Una jerarquía de directorio organizada ayuda a mantener el proyecto web estructurado y facilita la gestión del código y los recursos. Los directorios comunes incluyen 'index.html', 'css/', 'js/', y 'images/'.

  • II. Elementos de estructura para el diseño de una página Web: Concepto
    • 1. Regla utilizada para validar y certificar una página web

      Las páginas web deben ser validadas con herramientas como el validador W3C para asegurar que el código HTML y CSS cumpla con los estándares web y sea compatible con diferentes navegadores.

    • 2. Reglas para escribir HTML

      Las reglas para escribir HTML incluyen el uso correcto de etiquetas, la anidación adecuada, y la adherencia a la sintaxis del lenguaje para garantizar la estructura y el funcionamiento correcto de la página.
